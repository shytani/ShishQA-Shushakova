HTTP (Hyper Text Transfer Protocol)  Протокол Передачи Гипертекста. 
HTTP предлагает набор правил и стандартов, которые регулируют способ передачи любой информации во Всемирной паутине. 

HTTPS - это защищенный протокол передачи гипертекста (Hyper Text Transfer Protocol Secure). Это продвинутая и безопасная версия HTTP. 
Это комбинация протокола SSL и HTTP. Обеспечивает зашифрованную и безопасную идентификацию сетевого сервера.

Разница:

HTTP					
•	Hypertext Transfer Protocol 		
•	Порт по умолчанию – 80 		
•	http://				
•	Нет защиты передаваемой 
информации. Любой, кто 
прослушивает трафик может 
получить доступ к данным.
•	Не использует шифрование
•	Сайтам с HTTP не нужен SSL	

HTTPS
•	Hypertext Transfer Protocol Secure
•	Порт по умолчанию -  443
•	https://
•	HTTPS шифрует данные перед передачей их по сети. На стороне получателя, данные расшифровываются.
•	Данные шифруются
•	Для работы с HTTPS нужен SSL сертификат

Разница версий
В HTTP 1.0 соединение закрывается после одной пары запрос/ответ.
В HTTP/1.1 введён режим «постоянного соединения», при котором соединение можно повторно использовать для нескольких запросов. 
HTTP/2 в отличие от предыдущих версий, является бинарным вместо текстового, передача нескольких асинхронных HTTP-запросов по одному TCP-соединению,  сжатие заголовков методом HPACK, несколько ответов на один запрос.
